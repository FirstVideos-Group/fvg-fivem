Kliens exportok
exports['fvg-seatbelt']:IsBelted() → boolean
Client
Visszaadja, hogy a helyi játékosnak be van-e csatolva a biztonsági öve. Az fvg-vehiclehud és más scriptek ezzel szinkronizálhatják saját állapotukat.
Lua példa
-- Ellenőrzés más scriptből (pl. fvg-vehicledamage) local belted = exports['fvg-seatbelt']:IsBelted() if belted then -- játékos védve van end
exports['fvg-seatbelt']:SetBelted(state)
Client
Programozottan állítja be a biztonsági öv állapotát. Lejátssza a hangot és frissíti az fvg-vehiclehud modulját. Használható pl. jármű spawnolásnál vagy mentett állapot visszaállításakor.
ParaméterTípusLeírás
statebooleantrue = becsatol, false = kienged
Lua példa
-- Automatikus becsatolás jármű spawnnál exports['fvg-seatbelt']:SetBelted(true) -- Kényszeres kiengedés (pl. jármű megsemmisüléskor) exports['fvg-seatbelt']:SetBelted(false)
exports['fvg-seatbelt']:GetBeltState() → table
Client
Részletes állapotobjektumot ad vissza, tartalmazza az öv állapotát, hogy a játékos járműben van-e és az aktuális sebességet.
Visszatérési mezőTípusLeírás
belted boolean Be van-e csatolva az öv
inVehicle boolean Járműben van-e a játékos
speed number Aktuális sebesség km/h-ban
Lua példa
local state = exports['fvg-seatbelt']:GetBeltState() -- state.belted = true/false -- state.inVehicle = true/false -- state.speed = 87.4 (km/h)
Szerver exportok
exports['fvg-seatbelt']:GetPlayerBeltState(playerId) → boolean
Server
Szerver oldalról lekérdezi egy játékos biztonsági öv állapotát. Használható statisztikákhoz, admin rendszerekhez vagy egyéb szerver oldali logikához.
ParaméterTípusLeírás
playerIdnumberA játékos szerver ID-ja
Lua példa – szerver script
-- fvg-admin vagy egyéb szerver script local isBelted = exports['fvg-seatbelt']:GetPlayerBeltState(source) if not isBelted then -- figyelmeztetés, bírság stb. end

Ütközés esetén: kiesés ellenőrzés
Ha az fvg-vehiclehud nincs telepítve, a Config.VehicleHudIntegration = false beállítással kikapcsolható az integráció. Ugyanígy az fvg-notify is.
Konfigurálható integrációk (config.lua)
Config.VehicleHudIntegration = true -- fvg-vehiclehud öv modul frissítése Config.NotifyIntegration = true -- fvg-notify értesítések
Kiesés logika áttekintése
A rendszer két feltétel esetén vizsgálja a kiesés lehetőségét.
FeltételLeírás
speedCrashA jármű sebessége egy frame alatt CrashSpeedDropRatio alá esik (alapért. 65%) és az előző sebesség > EjectMinSpeed km/h
bodyCrash A karosszéria egészsége MinBodyHealthDrop-nál többet csökkent egy frame alatt
A kiesés esélyalapú – a EjectChanceScale és a pillanatnyi sebesség határozza meg, hogy megtörténik-e. Ez realisztikusabb viselkedést eredményez, mint a determinisztikus rendszerek.
